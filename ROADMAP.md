# Colombian Spanish SRS - Feature Prioritization Roadmap

*Last Updated: March 2025*

---

## How to Use This Roadmap

**Priority Levels:**
- **P0 (Critical)** - Must have for MVP launch. Blocks release if not done.
- **P1 (High)** - Important for competitive parity. Include in v1.0 or v1.1.
- **P2 (Medium)** - Differentiating features. Include in v1.2-v1.5.
- **P3 (Low)** - Nice to have. Future releases or post-PMF.

**Effort Levels:**
- **XS** - 1-2 days
- **S** - 3-5 days
- **M** - 1-2 weeks
- **L** - 2-4 weeks
- **XL** - 1-2 months

**Impact Levels:**
- **High** - Major user value, competitive advantage, or revenue driver
- **Medium** - Noticeable user value, expected feature
- **Low** - Incremental improvement

## Notion Import Instructions

### Step 1: Create Database
1. Create a new database in Notion
2. Use these properties:
   - **Name** (Title)
   - **Phase** (Select: MVP, Content Quality, Engagement, Advanced, Expansion, Scale)
   - **Priority** (Select: P0, P1, P2, P3)
   - **Effort** (Select: XS, S, M, L, XL)
   - **Impact** (Select: High, Medium, Low)
   - **Status** (Select: Not Started, In Progress, Review, Done)
   - **Owner** (Person)
   - **Notes** (Text)
   - **Region** (Select: All, Paisa, Costeno, Caleno, etc. - for content)

### Step 2: Create Views
1. **By Phase** - Group by Phase
2. **By Priority** - Group by Priority (P0 first)
3. **By Status** - Kanban board view
4. **Content Only** - Filter to content features, group by Region
5. **Engineering Only** - Filter to technical features

### Step 3: Import Data
Copy each row from the tables above as a new database entry
